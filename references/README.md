# references/

External **reference material** that informs how we build Suplaykart.

This is research and inspiration — context for decisions, not code or final
assets. Keep it organized so anyone can understand *why* the product looks and
behaves the way it does.

## What goes here

- **Competitor research** — screenshots and notes on similar supply / e‑commerce
  platforms.
- **Design inspiration** — UI references, moodboards, layout ideas.
- **Specs & briefs** — PDFs, client requirements, feature lists.
- **Links** — a curated `links.md` of useful articles, docs, and examples.

## Conventions

- Add a short note (a line of Markdown) explaining *why* each reference was saved.
- Don't confuse this with `old-html/` (our own legacy site) or `assets/` (our own
  brand assets) — `references/` is **third‑party / external** material only.
- Respect copyright: references are for internal guidance, not for shipping.
